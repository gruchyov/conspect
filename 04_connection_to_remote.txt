
      Основные команды

   1) локального нет а удалённый есть
                      скопировать путь через кнопку code
                      в командной строке cd Desktop/
                      git clone https:...
          это рабочая копия, если вносить в этот репозиторий какие то изменения
          их можно сразу отправлять на удалённый сервер в исходный репозиторий
          поскольку они уже автоматически связаны

   2) нет ни удалённого на локального репозитория
                     создать удалённый репозиторий
                     1 вариант - создать в нём файл и закоммитить
                     2 вариант - загрузить файл и закоммитить
                       репозиторий после этого не пустой, 
                       клонировать и работать
                     3 вариант создать на своём компьютере в папке файл                     
           3.1git init - создать репозиторий в папке
           3.2 закоммитить файл git commit -m "first commit"
                                -git branch -M main
           3.3 git remote add origin path - устанавливает в настройках 
                                            локального репозитория путь к 
                                            удалённому репозиторию
                                                  remote - удалённый
                                                  add - добавлять
                                                  origin - исходный
                                                  path - путь 
   git push -u origin master - отправляет локальный репозиторий на сервер
                    -u - автоматически устанавливает связь между удалённым 
                         и локальным репозиториями таким образом, чтобы
                         можно вызывать команду git push без дополнительных
                         параментров
                     master - имеющиеся пока одна ветка
   git remote -v - посмотреть путь к удалённому репозиторию, с которым 
                   cвязан локальный.
   git remote remove origin - удаление пути к удалённому репозиторию
                                                 remove - удалить, устранить
   и снова git remote add origin path

   3) локальный есть а удалённого нет
       -создать удалённый репозиторий
       -войти в локальный
       -подключение к удаленному репозиторию, например:
        git remote add origin https://github.com/gruchyov/spesial-skill.git
       -git branch -M main
       -отправить в удалённый всю информацию из локального:
        git push -u origin main
